3.Project Description

3.1.Overview

Topic:

The aim of our project is to create an automated plant watering system using Raspberry Pi and soil moisture sensors. This device focuses on plant care by ensuring plants receive the right amount of water at the right time, reducing the risk of over or under watering. The system consists of a soil moisture sensor placed in the plant’s soil. The Raspberry Pi will monitor soil moisture level and make decisions based on moisture thresholds. When the soil moisture drops, the arduino Raspberry Pi will automatically activate a water pump to deliver a precise amount of water to the plant. Ideally, the project will also include a user interface, accessible through a web application or mobile app, allowing the user to set moisture threshold, monitor plant health remotely, and receive alerts or notifications when intervention is required.

We hope that this system can improve plant health, and reduce manual plant care efforts. Users will benefit from a smart, automated solution that helps maintain healthy plants by providing water precisely when needed. Moreover, the project can be improved to support multiple plants, making it a scalable and user-friendly solution for both indoor and outdoor gardening.

Motivation:

This project is intended to solve the current problem in indoor gardening, as people usually forget to water their plants. The system will automate the process of watering plants, and keep track of the plant’s overall health. The project combines technology with a practical, real-world application that can address everyday challenges. It fits into current IT trends by embracing automation, sustainability, IoT connectivity, data-driven, and decision-making making it an interesting system for tech enthusiasts and plant lovers. This project will help us develop a variety of skills for our future career including technical skills, problem-solving abilities, project management competency, and a commitment to innovation and sustainability, etc. These skills are highly valued by potential employers, especially in IT and technology industries.

Landscape:

There are several similar systems and products that are available in the market, and it’s important to understand competition and compare differences with our own systems. The most common products are commercial automated plant watering systems. These systems are typically well designed and are easy to set up, however they can be relatively expensive. There are also IoT-based plant care devices which monitor plant health factors, including soil moisture, temperature, and light levels. These devices send data to mobile apps for remote monitoring but do not include any automated watering capabilities. Although there are alot of existing product and DIY solutions for automated plant watering systems, our project difference lies in its customization, scalability, cost effectiveness, and being an open-source project. We believe that we have an edge, and will be a valuable option for plant enthusiasts, and tech-savvy.

3.5.Scope and Limits
3.5.1.Scope

As for the making of this project we have a few features that we can deliver at the end of the project:

  1. Sensor Monitoring: Soil moisture sensors will collect data about the moisture content of the soil. This data will then be transmitted to the Raspberry Pi for processing.
  2. Threshold Analysis: The Raspberry Pi compares the collected moisture data with the programmed threshold. If moisture levels are below the threshold, the system proceeds to the next step.
  3. Watering Activation: Upon detection of low moisture, the Raspberry Pi automatically activates the water pump. Water is then supplied to the tree’s root zone through an irrigation system.
  4. Weather Considerations: The system can factor in weather forecasts to prevent unnecessary watering. If rain is predicted, the system postpones watering. If rain is predicted, the system will postpone watering to conserve water.
  5. Data Storage and Visualization: The collected data is then stored and can be accessed through a user interface. Users can analyze trends in soil moisture and adjust accordingly.
  6. Remote Control: Users are granted remote access to the system via either a user-friendly web interface or a dedicated mobile application. This enables real-time monitoring of soil moisture levels, reception of alerts, and even manually activating the watering process.
3.5.2.Limits

With those key features in making, we do have a feel limit that we are currently facing:

  1. Ensuring the physical condition of the machine to be usable in an acceptable time in different weather conditions.
  2. Having enough storage space for data for the machine to learn from past data while not making the machine too large.
  3. No way of activating or clearing if the water output is clogged or blocked, and having some issue in the consistency of the moisture metering in case of where the moisture sensor is proper moisture while the surrounding is not.
  4. Having problems with comparing the data from multiple sources for the machine to make the needed judgment.
